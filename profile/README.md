## 📄 Overview

<a href="#hazel">Hazel</a> is an automated file organization utility for Mac that watches designated folders and applies user-defined rules to files as they arrive — moving, renaming, color-labeling, and deleting files automatically based on their name, date, type, content, and any other attribute the rules specify.

The <a href="#hazel">folder watching rule system</a> is Hazel's core mechanism — the user defines which folders Hazel monitors and creates rules for each folder specifying what to do with files that match certain conditions. When a file appears in a watched folder matching the rule conditions, Hazel applies the configured actions automatically without any user intervention. A rule might move all PDF files containing the word 'invoice' to the Invoices folder, rename them with the date prefix, and apply a green color label — automatically processing every PDF that matches when it appears in the Downloads folder. The <a href="#hazel">condition variety</a> covers virtually every file attribute as a rule trigger — filename containing specific text, file extension, file kind, date created, date modified, file size, content matching patterns, color label, tags, and others — with conditions combining through AND and OR logic for precise rule targeting.

The <a href="#hazel">action variety</a> applies to matching files what the rule specifies — move to folder, copy to folder, rename with configurable naming patterns, apply color label, add tags, open with application, run AppleScript or shell script, delete, move to Trash, and many others. Multiple actions chain in a single rule for compound processing — match a PDF invoice, extract the date from the filename, rename the file with a standardized date format, move it to the correct year folder, and apply a label, all in a single automated sequence. The <a href="#hazel">Downloads folder management</a> is Hazel's most universally useful application — the Downloads folder accumulates files from every browser download, email attachment save, and application export. Hazel rules process this accumulation automatically, routing files to appropriate locations and cleaning up temporary downloads after a set period.</p>

<a href="#hazel">Trash management rules</a> automate Trash emptying — deleting Trash contents older than a configurable period, keeping Trash size below a limit, or emptying the Trash on a schedule without requiring manual Trash management. <a href="#hazel">App sweep for application removal</a> detects when applications move to Trash and offers to remove associated preference files, caches, and support files — the thorough application removal that dragging to Trash alone doesn't accomplish.

<a href="#hazel">Import folders for media management</a> automatically imports and organizes photos, music, and other media files — routing files from an import folder to the correct library location, triggering imports into Photos or Music, or organizing files by date into a folder hierarchy without manual processing. <a href="#hazel">Rule previewing</a> tests rules against the folder contents before activating them — showing which files would match and what actions would apply without actually executing the actions, for confirming rule behavior before committing.</p>

`File organization` · `Mac automation` · `Downloads management` · `Paperless workflow` · `File management` · `Folder watching` · `Mac utilities` · `System maintenance` · `Document organization` · `Productivity automation`

Hazel's rule system represents the right abstraction level for file automation — specific enough to precisely target intended files but general enough to handle the variety that real file systems contain. The combination of filename, date, type, size, and content conditions with AND and OR logic covers the full range of real file organization needs: invoice PDFs (content: 'invoice' AND extension: pdf), downloads older than a week (kind: not folder AND date: older than 7 days), screenshots accumulating on the Desktop (name: starts with 'Screen Shot' AND kind: image). The specificity prevents unintended file moves that simpler automation would create.

## ❓ FAQ

<details>
<summary>What does Hazel do?</summary>

Hazel watches designated folders and applies user-defined rules to files as they arrive — moving, renaming, color-labeling, tagging, and deleting files automatically based on their attributes.

</details>

<details>
<summary>What conditions can Hazel rules use?</summary>

Filename content, file extension, file kind, date created, date modified, file size, Spotlight content matching, color label, tags, and many others — combinable with AND and OR logic.

</details>

<details>
<summary>What actions can Hazel rules apply?</summary>

Move to folder, copy, rename, apply color label, add tags, open with application, run AppleScript or shell script, delete, move to Trash, and many others in sequence within a single rule.

</details>

<details>
<summary>Can Hazel manage the Trash automatically?</summary>

Yes. Trash management rules can empty Trash on a schedule, after a set age, or when size exceeds a limit.

</details>

<details>
<summary>Does Hazel remove application support files when I delete an app?</summary>

Yes. App Sweep detects applications moved to Trash and offers to remove associated preference files, caches, and support files for thorough removal.

</details>

<details>
<summary>Can I test Hazel rules before activating them?</summary>

Yes. Rule previewing shows which files in the folder would match and what actions would apply without executing the actions.

</details>

<details>
<summary>Does Hazel support AppleScript?</summary>

Yes. AppleScript and shell script actions extend Hazel rules with any custom automation beyond the built-in actions.

</details>

<details>
<summary>How many folders can Hazel watch simultaneously?</summary>

Hazel monitors as many folders as needed, each with independent rule sets.

</details>

<details>
<summary>Does Hazel run on Apple Silicon?</summary>

Yes. Current versions support Apple Silicon Macs natively.

</details>
